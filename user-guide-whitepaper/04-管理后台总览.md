# 04 管理后台总览

## Admin 是什么

Admin 是管理员后台。管理员通过它管理整个 authentik。

当前项目后台侧边栏分为以下模块：

| 模块 | 主要页面 | 白话理解 |
| --- | --- | --- |
| Dashboards | Overview、User Statistics、System Tasks | 看系统健康和运行情况 |
| Applications | Applications、Providers、Outposts | 接入业务应用 |
| Endpoint Devices | Devices、Device access groups、Connectors | 管理端点设备和连接器 |
| Events | Logs、Notification Rules、Notification Transports 等 | 查日志、配通知、做审计 |
| Customization | Policies、Property Mappings、Blueprints、Files、Reputation scores | 自定义规则、字段和配置 |
| Flows and Stages | Flows、Stages、Prompts | 管登录、注册、MFA 等流程 |
| Directory | Users、Groups、Roles、Sources、Tokens、Invitations | 管账号、组、权限和外部身份源 |
| System | Brands、Certificates、Outpost Integrations、Settings | 管系统级配置 |
| Enterprise | Licenses | 企业版许可证 |

## 新手进入后台先看哪里

建议顺序：

1. `Dashboards -> Overview`：看系统是否正常。
2. `Directory -> Users`：看用户。
3. `Directory -> Groups`：看用户组。
4. `Applications -> Applications`：看已接入应用。
5. `Applications -> Providers`：看协议配置。
6. `Events -> Logs`：看最近错误。

## 后台模块之间的关系

最常见的关系是：

1. 用户在 Directory 里。
2. 用户属于组。
3. 应用在 Applications 里。
4. 应用关联 Provider。
5. 应用绑定用户、组或策略。
6. 登录过程由 Flow 和 Stage 控制。
7. 问题记录在 Events。

## 管理后台的使用习惯

建议每次做配置时都按这个习惯：

1. 先确认目标是什么。
2. 找到对应模块。
3. 修改前记下当前配置。
4. 修改后用测试用户验证。
5. 出问题先看 Events。

## 不建议新手一开始做什么

- 不要直接大改默认登录 Flow。
- 不要给所有用户开放高敏感应用。
- 不要随意删除证书。
- 不要把所有权限都绑到单个用户。
- 不要忽略 Events 中的错误信息。
