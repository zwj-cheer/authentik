# 00 阅读说明

## 这套文档适合谁

适合下面几类人：

- 第一次接触 authentik。
- 不清楚 Application、Provider、Flow、Stage、Policy 分别是什么。
- 需要管理用户、组、应用和访问权限。
- 需要排查登录失败、授权失败、应用看不到等问题。
- 需要接手当前项目，但还不熟悉后台菜单。

## 这套文档怎么读

不要一上来就看所有高级配置。建议先按这个顺序理解：

1. authentik 是统一登录中心。
2. 用户通过 Flow 登录。
3. 登录后用户在 User 界面看到应用。
4. 管理员在 Admin 界面管理用户、应用、协议和策略。
5. Application 是用户看到的应用入口。
6. Provider 是应用背后的登录协议配置。
7. Policy、User、Group Binding 决定谁能访问。
8. Events 是排查问题的第一入口。

## 本文档不做什么

本文档不讲源码实现细节，不教你改代码，也不替代官方完整文档。它只解决一个问题：让小白能理解当前项目后台每个主要模块是干什么的，并能开始管理系统。

## 管理时的基本原则

- 先用测试用户验证，再影响真实用户。
- 先按组授权，少给单个用户单独开权限。
- 先看 Events，再猜原因。
- 高风险应用用更严格策略。
- 不确定时先截图或记录旧配置，再修改。
