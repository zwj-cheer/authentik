# 17 常见问题排查手册

## 排查原则

先看日志，再改配置。

authentik 的大多数问题都能从 `Events -> Logs` 找到线索。

## 问题 1：用户登录失败

可能原因：

- 密码错误。
- MFA 失败。
- 用户被禁用。
- Source 登录异常。
- Flow 或 Stage 配置错误。
- Policy 拒绝访问。

排查步骤：

1. 进入 `Events -> Logs`。
2. 搜索用户名。
3. 找到失败事件。
4. 看失败在哪个 Flow 或 Stage。
5. 检查对应配置。

## 问题 2：用户登录成功但看不到应用

可能原因：

- 应用没有授权给用户。
- 用户不在对应用户组。
- Application 的 Policy engine mode 太严格。
- Binding 没启用。
- Policy 判断失败。

排查步骤：

1. 查看 Application 的绑定。
2. 检查用户所属 Groups。
3. 检查绑定是否 Enabled。
4. 如果是 ALL，逐个检查每个条件。
5. 看 Events 中是否有访问拒绝事件。

## 问题 3：点击应用后跳转失败

可能原因：

- Provider 配置错误。
- Redirect URI 错误。
- ACS URL 错误。
- Launch URL 错误。
- 目标应用配置不匹配。

排查步骤：

1. 查看应用关联的 Provider。
2. 对照目标应用配置。
3. 检查回调地址。
4. 检查 Events。
5. 用测试用户重新触发。

## 问题 4：OIDC 登录失败

重点检查：

- Client ID。
- Client Secret。
- Redirect URIs。
- Client Type。
- Scopes。
- Signing Key。
- Token 有效期。

最常见问题：

Redirect URI 和目标应用不一致。

## 问题 5：SAML 登录失败

重点检查：

- ACS URL。
- Issuer / Entity ID。
- Audience。
- Signing Certificate。
- Verification Certificate。
- NameID Mapping。
- Assertion 有效时间。

最常见问题：

- ACS URL 填错。
- Entity ID 不匹配。
- 证书过期。
- NameID 不符合目标应用要求。

## 问题 6：Proxy 应用打不开

重点检查：

- External host。
- Internal host。
- Outpost 是否在线。
- Cookie domain。
- DNS。
- TLS。
- Unauthenticated Paths。

常见错误：

把用户访问地址和内部上游地址填反。

## 问题 7：用户组授权不生效

可能原因：

- 用户没有加入组。
- 绑定的是错误组。
- Application 的 Group 展示字段被误认为权限组。
- Binding 未启用。
- ALL 模式下其他条件没通过。

处理：

1. 确认用户实际属于该组。
2. 确认应用绑定的是该组。
3. 检查 Policy engine mode。
4. 看 Events。

## 问题 8：修改 Flow 后登录异常

可能原因：

- Stage 顺序错误。
- 缺少必要 Stage。
- Policy 阻断。
- 默认 Brand 指向错误 Flow。

处理：

1. 回到 `Flows and Stages -> Flows`。
2. 查看 Flow 图。
3. 检查 Stage Binding。
4. 临时切回原 Flow。
5. 用测试 Flow 重做修改。

## 问题 9：外部身份源同步异常

重点检查：

- Source 网络连通。
- 凭据是否正确。
- Base DN。
- 用户过滤器。
- 组过滤器。
- 属性映射。
- 删除策略。

建议：

先小范围同步测试，不要一开始同步全部用户。

## 问题 10：证书相关错误

表现：

- SAML 登录失败。
- 签名校验失败。
- 加密断言失败。
- 浏览器提示证书错误。

处理：

1. 进入 `System -> Certificates`。
2. 检查证书是否过期。
3. 检查 Provider 是否引用正确证书。
4. 更新目标应用中的证书。
5. 重新测试。
