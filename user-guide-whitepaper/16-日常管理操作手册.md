# 16 日常管理操作手册

## 操作 1：创建一个普通用户

适用场景：新增员工、测试账号、外部协作者。

步骤：

1. 进入 `Directory -> Users`。
2. 点击创建用户。
3. 填写用户名、姓名、邮箱等信息。
4. 设置密码或让用户通过邀请完成。
5. 保存。
6. 把用户加入对应 Groups。
7. 用测试方式确认用户能登录。

检查点：

- 用户名是否唯一。
- 邮箱是否正确。
- 是否加入正确用户组。
- 是否不小心给了管理员权限。

## 操作 2：创建用户组

适用场景：按部门、角色或系统权限管理用户。

步骤：

1. 进入 `Directory -> Groups`。
2. 创建组。
3. 填写组名。
4. 如有需要设置父级组。
5. 添加用户。
6. 后续在应用绑定中使用这个组。

建议命名：

- `研发部`
- `运维组`
- `财务组`
- `全员`
- `外部协作者`

## 操作 3：接入一个 OIDC 应用

步骤：

1. 进入 `Applications -> Applications`。
2. 启动应用创建向导。
3. 第 1 页填写应用名称、Slug、展示组。
4. 第 2 页选择 OAuth2/OIDC Provider。
5. 第 3 页配置 Client Type、Redirect URI、Authorization Flow。
6. 复制 Client ID 和 Client Secret 到目标应用。
7. 第 4 页绑定测试用户组。
8. 第 5 页检查并提交。
9. 用测试用户访问应用。

重点：

- Redirect URI 必须一致。
- Client Secret 不能泄漏。
- 先用测试用户组验证。

## 操作 4：接入一个 SAML 应用

步骤：

1. 确认目标应用支持 SAML。
2. 从目标应用拿到 ACS URL、Entity ID 等信息。
3. 创建 SAML Provider。
4. 填写 ACS URL、Issuer、Audience。
5. 配置 Signing Certificate。
6. 配置 NameID Property Mapping。
7. 创建 Application 并关联 Provider。
8. 配置访问绑定。
9. 把 authentik 的元数据或证书填回目标应用。
10. 测试登录。

重点：

- ACS URL 和 Entity ID 最容易错。
- 证书要有效。
- NameID 要符合目标应用要求。

## 操作 5：给应用授权

推荐方式：按用户组授权。

步骤：

1. 进入应用详情或创建向导的绑定页面。
2. 添加 Binding。
3. 选择 Group。
4. 选择目标用户组。
5. Enabled 保持开启。
6. 保存。
7. 确认 Policy engine mode 是否符合预期。
8. 用组内测试用户访问。

## 操作 6：收紧高敏感应用访问

适合财务、运维、后台、生产系统。

推荐组合：

- 绑定管理员或运维组。
- 绑定 MFA 策略。
- 绑定内网或可信网络策略。
- Policy engine mode 选择 ALL。

测试顺序：

1. 先确认用户组可以访问。
2. 再加 MFA 策略。
3. 再加网络策略。
4. 最后切到 ALL。

## 操作 7：查看登录失败原因

步骤：

1. 进入 `Events -> Logs`。
2. 搜索用户名。
3. 找到失败事件。
4. 查看失败原因和关联对象。
5. 根据失败点检查 Flow、Stage、Policy、Provider 或 Source。

常见失败点：

- 密码错误。
- MFA 失败。
- 用户不在授权组。
- Redirect URI 错误。
- SAML 证书错误。
- Source 同步异常。

## 操作 8：修改登录页品牌

步骤：

1. 进入 `System -> Brands`。
2. 找到默认 Brand 或对应域名 Brand。
3. 修改标题、Logo、Favicon 等。
4. 检查默认 Authentication Flow。
5. 保存后用无痕窗口测试登录页。

注意：

不要随意切换默认认证 Flow，除非你已经测试过。

## 操作 9：更新证书

步骤：

1. 进入 `System -> Certificates`。
2. 查看即将过期的证书。
3. 创建或导入新证书。
4. 找到使用旧证书的 Provider。
5. 切换到新证书。
6. 测试登录。
7. 确认无引用后再删除旧证书。

## 操作 10：用 Blueprint 导入配置

步骤：

1. 先在测试环境准备 Blueprint。
2. 进入 `Customization -> Blueprints`。
3. 导入 Blueprint。
4. 查看执行结果。
5. 检查创建了哪些对象。
6. 用测试用户验证。

注意：

不要直接在正式环境导入不理解的 Blueprint。
