# 01 项目总览

## authentik 是什么

authentik 是一个开源身份提供商，也就是 IdP。你可以把它理解为“统一登录中心”。

它的核心作用是：

1. 让用户只登录一次。
2. 让多个业务系统共享同一套登录入口。
3. 让管理员统一管理用户、组、权限、登录流程和审计日志。

## 它解决什么问题

如果没有 authentik，每个业务系统都要自己管理账号、密码、权限和登录安全。系统一多，问题会越来越多：

- 用户要记很多账号密码。
- 离职用户可能没有及时从所有系统删除。
- 每个系统的 MFA、密码策略、审计日志都不一致。
- 管理员很难知道用户到底访问了哪些系统。

authentik 把这些事情集中起来管理。

## 当前项目包含哪些部分

从当前仓库可以看出，这个项目不是单一语言项目，而是多个部分协作：

| 部分 | 作用 |
| --- | --- |
| Python/Django | 核心后端、API、模型、流程、策略、事件 |
| TypeScript/Lit Web UI | 管理后台、用户门户、登录流程页面 |
| Go 组件 | server、proxy、ldap、radius、rac 等运行组件 |
| Rust 组件 | 项目中的部分底层能力 |
| Docker | 容器构建和部署 |
| website/docs | 官方文档站点 |
| blueprints | 默认配置和可导入配置 |

## 三个核心界面

| 界面 | 谁使用 | 作用 |
| --- | --- | --- |
| Flow | 所有登录用户 | 执行登录、注册、找回密码、MFA、授权确认等流程 |
| User | 普通用户 | 查看可访问应用，管理个人设置、MFA、令牌 |
| Admin | 管理员 | 管理用户、组、应用、协议、流程、策略、事件和系统配置 |

## 新手最重要的理解

authentik 不是单纯的“登录页面”。它是一套身份控制系统。

用户访问应用时，通常会经历：

1. 用户点击应用。
2. 应用把用户跳转到 authentik。
3. authentik 执行登录 Flow。
4. authentik 检查用户、组和策略。
5. authentik 通过 Provider 和应用完成协议交互。
6. 用户进入目标应用。

## 管理员日常关注什么

管理员最常做这些事：

- 创建用户和组。
- 接入新应用。
- 给应用配置 Provider。
- 配置谁能访问应用。
- 修改登录流程。
- 查看事件日志。
- 排查登录失败。
- 管理证书、品牌和系统设置。
