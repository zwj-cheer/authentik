# 03 登录与用户门户：Flow 和 User

## Flow 界面是什么

Flow 是用户登录时看到的流程界面。

它可能出现在这些场景：

- 用户登录 authentik。
- 用户访问某个应用，被跳转到 authentik 认证。
- 用户注册账号。
- 用户找回密码。
- 用户配置 MFA。
- 用户授权某个应用访问自己的信息。

## Flow 对小白意味着什么

你可以把 Flow 想成一张“流程表”。

每个流程由多个 Stage 组成。用户按顺序完成每个 Stage，最后得到结果。

例如登录流程：

1. 识别用户是谁。
2. 输入密码。
3. 做 MFA。
4. 登录成功。

## User 界面是什么

User 是普通用户的应用门户。

用户登录后可以在这里看到自己能访问的应用，也可以管理自己的个人设置。

当前项目的 User 路由主要包括：

| 页面 | 作用 |
| --- | --- |
| `/library` | 应用库，展示用户可访问应用 |
| `/settings` | 用户设置 |

## 用户设置通常包括什么

从当前项目文件可以看到，用户设置包含：

- 密码相关设置。
- MFA 设备管理。
- 用户令牌管理。
- 用户设置 Flow 执行页面。

## 普通用户怎么使用

普通用户一般只需要：

1. 打开用户入口。
2. 登录。
3. 在应用库里找到目标应用。
4. 点击应用进入业务系统。
5. 必要时进入设置页面管理 MFA 或令牌。

## 管理员需要关注什么

管理员要确保：

- 用户能完成登录 Flow。
- 用户能看到应有的应用。
- 用户不该看到的应用不会出现。
- 用户设置 Flow 不会阻塞正常使用。
- MFA、密码、令牌配置符合安全要求。

## 常见问题

| 问题 | 可能原因 | 排查入口 |
| --- | --- | --- |
| 用户登录不了 | Flow、Stage、密码、MFA、Source 异常 | Events -> Logs |
| 用户看不到应用 | 应用绑定、用户组、策略不匹配 | Applications、Policies、Users |
| 用户点击应用失败 | Provider 配置或回调地址错误 | Provider 配置、Events |
| 用户无法改密码 | 用户设置 Flow 或权限问题 | Flows、Users、Events |
