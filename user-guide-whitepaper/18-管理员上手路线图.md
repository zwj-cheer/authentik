# 18 管理员上手路线图

## 第 1 天：看懂系统

目标：

- 知道 authentik 是统一登录中心。
- 知道 Flow、User、Admin 三个界面。
- 知道后台主要菜单在哪里。

要做：

1. 登录 Admin。
2. 查看 `Dashboards -> Overview`。
3. 查看 `Directory -> Users`。
4. 查看 `Directory -> Groups`。
5. 查看 `Applications -> Applications`。
6. 查看 `Events -> Logs`。

完成标准：

- 知道系统状态在哪里看。
- 知道用户在哪里管理。
- 知道应用在哪里管理。
- 知道问题先看 Events。

## 第 2 天：管理用户和组

目标：

- 能创建用户。
- 能创建用户组。
- 能把用户加入组。

要做：

1. 创建测试用户。
2. 创建测试用户组。
3. 把测试用户加入测试组。
4. 用测试用户登录。

完成标准：

- 测试用户能登录。
- 测试用户属于正确组。

## 第 3 天：接入一个测试应用

目标：

- 理解 Application 和 Provider。
- 能创建一个应用入口。
- 能配置访问绑定。

要做：

1. 创建 Application。
2. 选择 Provider 类型。
3. 配置 Provider。
4. 绑定测试用户组。
5. 用测试用户访问。

完成标准：

- 测试用户能在应用库看到应用。
- 点击应用后能进入目标应用或看到明确错误。
- 能从 Events 看懂失败原因。

## 第 4 天：理解权限和策略

目标：

- 理解 Group Binding。
- 理解 Policy Binding。
- 理解 ANY 和 ALL。

要做：

1. 给应用绑定用户组。
2. 测试 ANY 模式。
3. 增加一个策略。
4. 测试 ALL 模式。

完成标准：

- 能解释为什么某个用户可以或不可以访问应用。

## 第 5 天：理解 Flow 和 Stage

目标：

- 知道登录流程由多个 Stage 组成。
- 能看懂默认登录 Flow。

要做：

1. 打开 `Flows and Stages -> Flows`。
2. 查看默认认证 Flow。
3. 查看绑定的 Stages。
4. 不修改正式 Flow，只做阅读和测试。

完成标准：

- 知道登录卡在哪一步时该看哪个 Stage。

## 第 6 天：学习排查

目标：

- 能定位常见登录和授权问题。

要做：

1. 人为制造一次测试用户访问失败。
2. 在 Events 中找到失败事件。
3. 判断是用户组、策略、Provider 还是 Flow 问题。

完成标准：

- 遇到问题不会先乱改配置，而是先看日志。

## 第 7 天：进入日常管理节奏

目标：

- 能独立完成基本管理员工作。

日常清单：

- 每天看 Overview 和 Events。
- 新用户先加入组。
- 新应用先测试后开放。
- 高敏感应用使用 ALL 和 MFA。
- 定期检查证书和 Token。
- 定期检查管理员组成员。

## 最终能力标准

看完并练习后，你应该能：

- 解释 authentik 的基本作用。
- 创建用户和组。
- 接入一个应用。
- 选择合适 Provider。
- 配置应用访问权限。
- 理解 ANY 和 ALL。
- 看懂 Flow 和 Stage。
- 用 Events 排查问题。
- 知道哪些配置属于高风险修改。
